## 二、隐私安全评估

### 2.1 敏感数据脱敏机制

#### ✅ MAC地址脱敏
- **脱敏效果**: `AA:BB:CC:DD:EE:FF` → `已脱敏`
- **哈希生成**: HMAC-SHA256 哈希值正确生成
- **哈希长度**: 64字符（SHA-256标准）
- **确定性**: 相同MAC地址生成相同哈希
- **唯一性**: 不同MAC地址生成不同哈希
- **双重脱敏**: 多次脱敏结果一致

**验证代码**:
```python
mac = "AA:BB:CC:DD:EE:FF"
hash1 = hash_mac_address(mac)
hash2 = hash_mac_address(mac)
assert hash1 == hash2  # ✅ 确定性
assert len(hash1) == 64  # ✅ SHA-256长度
```

#### ✅ 序列号脱敏
- **主板序列号**: `MB-SERIAL-123456` → `已脱敏`
- **BIOS序列号**: `BIOS-SERIAL-789012` → `已脱敏`
- **结构保留**: 非敏感字段保持原值

#### ✅ 脱敏配置
```python
SENSITIVE_FIELDS = frozenset({"serial_number", "mac_address"})
```
- 使用 `frozenset` 防止运行时修改
- 集中管理敏感字段定义

### 2.2 数据加密机制

#### ✅ HMAC-SHA256 实现
- **算法**: HMAC-SHA256（RFC 2104）
- **盐值**: `system-info-collector-v2-salt`
- **密钥长度**: 28字节
- **输出**: 64字符十六进制字符串

**验证**:
```python
expected = hmac.new(SALT, mac.encode(), hashlib.sha256).hexdigest()
actual = hash_mac_address(mac)
assert expected == actual  # ✅ 实现正确
```

#### ✅ 安全文件删除
- **覆写机制**: 使用 `\x00` 字节覆写原文件内容
- **同步刷新**: `fsync()` 确保写入磁盘
- **删除验证**: 覆写后删除文件
- **异常处理**: 覆写失败时尝试直接删除

**测试场景**:
- 正常文件删除: ✅ 通过
- 不存在文件: ✅ 安全返回 False

#### ✅ 脚本完整性校验
- **算法**: SHA-256
- **存储**: `.ps1.sha256` 文件
- **校验时机**: 每次执行前验证
- **篡改检测**: 哈希不匹配时拒绝执行

#### ✅ 原子写入机制
- **临时文件**: `.tmp` 后缀
- **文件锁**: msvcrt 锁定保护
- **同步刷新**: `fsync()` 确保持久化
- **重命名**: 原子替换目标文件
- **清理**: 失败时删除临时文件

### 2.3 访问权限控制

#### ✅ 导出路径验证
- **允许路径**: 
  - 缓存目录 (`CACHE_DIR`)
  - 当前工作目录 (`cwd`)
  - 用户主目录 (`home`)
- **拒绝路径**: 
  - `C:\Windows\System32\*` ✅
  - 系统目录 ✅
  - 其他受限目录 ✅

**实现**:
```python
def _validate_export_path(output_path: str) -> Optional[str]:
    resolved = Path(output_path).resolve()
    return _validate_path(resolved, [
        CACHE_DIR.resolve(), 
        Path.cwd().resolve(), 
        Path.home().resolve()
    ])
```

#### ✅ 对比路径验证
- **限制范围**: 仅允许缓存目录内文件
- **格式检查**: 必须为 `.json` 扩展名
- **存在性检查**: 文件必须存在
- **路径遍历保护**: 使用 `is_relative_to()` 验证

#### ✅ 导入路径验证
- **格式验证**: 仅接受 `.json` 文件
- **内容验证**: 必须为 JSON 对象
- **字段验证**: 检查必需字段（采集时间/计算机名）
- **异常处理**: 完整捕获并提示

#### ✅ 路径遍历保护
- **解析路径**: 使用 `Path.resolve()` 规范化
- **相对路径检查**: `is_relative_to()` 验证
- **前缀碰撞防护**: 防止 `cache_dir` vs `cache_dir_evil`

### 2.4 异常数据隐私保护

#### ✅ 错误日志脱敏
- **日志内容**: 不包含敏感字段值
- **异常信息**: 仅记录错误类型和位置
- **调试模式**: 即使 DEBUG 级别也不泄露敏感数据

**验证**:
```python

assert "serial_number" not in log_output  # ✅
assert "mac_address" not in log_output    # ✅
```

#### ✅ 损坏数据处理
- **自动检测**: JSON 解析失败时识别
- **安全删除**: 自动删除损坏文件
- **不泄露**: 损坏内容不输出到日志

#### ✅ 无效输入处理
- **空配置**: `query_config({}, "any")` → `None`
- **None值**: `query_config({"CPU": None}, "CPU.name")` → `None`
- **异常类型**: 不抛出异常，安全返回

#### ⚠️ P1: 缓存文件权限设置（中优先级）
**现状**: 缓存文件使用默认权限创建  
**风险**: 其他用户可能读取缓存文件  
**建议**: 
```python
import stat
os.chmod(CACHE_FILE, stat.S_IRUSR | stat.S_IWUSR)  # 仅所有者可读写
```
**影响**: 多用户环境下的隐私保护

#### ⚠️ P2: 敏感字段配置化（中优先级）
**现状**: 敏感字段硬编码在常量中  
**建议**: 支持从配置文件读取敏感字段列表  
**好处**: 用户可自定义脱敏规则

#### ⚠️ P3: 日志脱敏过滤器（低优先级）
**现状**: 手动确保日志不包含敏感数据  
**建议**: 实现日志过滤器自动脱敏  
**实现**:
```python
class SensitiveDataFilter(logging.Filter):
    def filter(self, record):
        record.msg = _anonymize_string(record.msg)
        return True
```
